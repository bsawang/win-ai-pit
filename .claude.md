# Windows 避坑指南

AI 驱动的 Windows 开发/运维踩坑知识库。

## 使用流程

当用户遇到 Windows 相关问题（编码、注册表、MSYS、提权等）：

1. **先查** — 调用 `search_pitfall` 搜索已知坑
2. **如果命中** — 直接给出解决方案
3. **如果没命中** — 比对本地已有条目（去重检测）
4. **确实没有** — git fetch 检查远程，有变化才 pull + 重建索引 + 重搜
5. **还是没有** — 调用 `record_pitfall` 记录新坑（写 .md + 重建索引 + git commit + push）

注意：只在问题**解决后**一次性记录（症状+根因+方案全），不记"待解决"坑。安装步骤见 [README](README.md)。

### search_pitfall
搜索已知坑。参数：query, tool, os, severity, limit。

### record_pitfall
记录新坑。自动去重（标题/症状相似度匹配），写文件后自动重建索引。
参数：title, symptom, root_cause, solution, os, tool, tool_versions, severity, tags。
